# AlgoBlock

## 项目介绍

AlgoBlock 是一款声明式序列变换谜题游戏。玩家通过嵌套组合“积木”编写表达式，将给定的整数序列变换为目标序列，从而完成关卡挑战。

### 游戏机制

游戏的核心玩法是将复杂的算法逻辑封装成一个个“原子积木”。玩家可以在特定的关卡中，调用内置或由 MOD 制作的扩展积木。这些积木具有特定的名称、参数列表、数据处理逻辑以及多种输出形式。

为了保证统一的接口与游戏体验，我们主要聚焦于一类线性变换的基础算法，通过指令嵌套的方式，在充满极客风格的光影特效终端里，体验“一行指令解决复杂问题”的乐趣。

### 技术特性

- **数据与渲染解耦**：核心算法逻辑（“怎么算”）与视觉呈现（“怎么画”）完全分离。编写新的逻辑积木无需了解底层 OpenGL 渲染，极大降低了扩展门槛。
- **MOD 系统**：游戏具有高度的扩展性。MOD 引擎提供标准化接口，开发者可以自由编写自定义库，实现从谜面到谜底的完整关卡设计。
- **极客风渲染管线**：灵感来源于《黑客帝国》与《Hacknet》等作品。采用定制化的渲染工具链，支持流畅的光标残影特效及其他炫酷视觉反馈，带来沉浸式的终端操作体验。
- **模块化设计**：项目采用模块化和高度解耦的架构，利用依赖注入与反射机制，确保了代码的规范性与优良的工程质量。

## 快速开始

### 基础开发环境要求

- **JDK 21** 或更高版本
- 与 JDK 版本匹配的 **Gradle**
- 支持 OpenGL 的显卡 (大部分现代显卡都支持 OpenGL)

> [!NOTE]
>
> 开发机使用的构建配置与部分重要依赖版本参考:
>
> Gradle 会在首次运行时自动下载相关依赖。 (需要良好的网络环境)
>
> - **Java 版本**: JDK 21
> - **Gradle 版本**: 8.9
> - **Shadow 插件版本**: 9.3.1
> - **图形库版本**: LWJGL 3.3.4

### 运行与构建

项目使用 Gradle 进行构建。首次运行时会自动下载相关的依赖。(需要良好的网络环境)

```bash
# 1. 克隆项目并进入目录
git clone https://github.com/Churk-Ben/AlgoBlock.git
cd AlgoBlock

# 2. 完整构建（自动下载 Gradle 及依赖）
./gradlew clean build

# 3. 运行游戏图形界面
./gradlew :game-gl:run
```

> [!NOTE]
>
> 如果网络环境不稳定, 你也可以选择手动下载依赖并进行构建:
>
> 1. 正确配置适用您当前系统的 JDK 和 Gradle.
> 2. 使用 Gradle 构建项目:
>
> ```bash
> gradle clean build
> ```
>
> 3. 运行游戏图形界面:
>
> ```bash
> gradle :game-gl:run
> ```

### 常用构建命令

| 操作         | 命令                           | 说明                                |
| ------------ | ------------------------------ | ----------------------------------- |
| **完整构建** | `./gradlew clean build`        | 清理并编译整个项目                  |
| **运行测试** | `./gradlew test`               | 运行所有单元测试                    |
| **启动游戏** | `./gradlew :game-gl:run`       | 直接启动图形界面进行调试            |
| **打包程序** | `./gradlew :game-gl:shadowJar` | 生成包含所有依赖的可执行 `jar` 文件 |

> 使用 `shadowJar` 打包成功后，可通过以下命令独立运行游戏：
>
> `java -jar game-gl/build/libs/game-gl-0.1.0-all.jar`
